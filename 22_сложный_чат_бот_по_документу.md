# Чат-бот по документу

## Уровень
Сложный

## Описание задачи
Нужно построить чат-бота, который отвечает на вопросы по текстовому документу или набору документов. Для учебной реализации можно сделать RAG-пайплайн: разбиение текста на фрагменты, поиск релевантных фрагментов и генерация ответа.

## Датасет
Hugging Face Datasets: [SQuAD v2](https://huggingface.co/datasets/rajpurkar/squad_v2)

SQuAD v2 содержит контексты, вопросы, ответы и примеры вопросов без ответа. Его удобно использовать для проверки QA-системы. Для демонстрации также можно взять свой PDF или текстовый документ.

## Что нужно получить
Система должна принимать вопрос пользователя и возвращать ответ, основанный на найденном фрагменте документа. Если ответа в документе нет, система должна уметь сообщить об этом.

## Примерные шаги реализации
1. Загрузить SQuAD v2 или подготовить собственный документ.
2. Разработать способ деления документа на фрагменты и обосновать его.
3. Представить фрагменты в виде, удобном для поиска по смыслу.
4. Реализовать поиск релевантных фрагментов для вопроса.
5. Подобрать количество фрагментов, которые будут использоваться для ответа.
6. Выбрать способ получения ответа из найденного контекста.
7. Вернуть ответ и ссылку на использованный фрагмент.
8. Проверить систему на вопросах, где ответ есть и где ответа нет.
9. Оценить качество вручную или по метрикам exact match/F1 для SQuAD.
10. Описать ограничения решения.

## Что должно получиться
Прототип чат-бота по документу с поиском источников и обработкой случаев, когда ответа нет.

## Возможные улучшения
- Проверить дополнительную переоценку найденных фрагментов.
- Сравнить разные размеры чанков.
- Добавить цитирование источника ответа.
- Сделать простой веб-интерфейс.
