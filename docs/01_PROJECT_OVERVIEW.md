# Обзор проекта ShkereBERT

## Цель
Построить **production-ready RAG-систему** для ответа на вопросы по документам с:
- цитированием источников;
- честной обработкой «нет ответа» (калиброванный отказ);
- поддержкой русского и английского языков;
- полной работой на CPU (on-premise, без GPU);
- воспроизводимыми метриками качества.

## Стек технологий

| Уровень | Технология | Версия | Зачем |
|---|---|---|---|
| Эмбеддинги | `sentence-transformers` (MiniLM) | 3.0.1 | Быстрые 384-мерные векторы на CPU |
| Векторный поиск | `faiss-cpu` (IndexFlatIP) | 1.8.0 | Точный cosine-поиск, миллисекунды на 10k чанков |
| Экстрактивный ридер | `transformers` (tinyroberta-squad2) | 4.41.2 | BERT-span + null-score, SQuAD v2 качество |
| Генеративный ридер | `llama-cpp-python` (Qwen 2.5 3B GGUF Q4) | 0.2.90 | Локальная LLM, офлайн, RU, RU/EN |
| Гибридный поиск | `rank-bm25` (BM25Okapi + RRF) | 0.2.2 | Ловит точные термины/номера |
| Ранжирование | `CrossEncoder` (ms-marco-MiniLM-L-6-v2) | — | Переоценка топ-кандидатов (опционально) |
| Интерфейсы | `streamlit` + `typer` | 1.35 / 0.12 | Веб-UI и CLI |
| Оценка | `evaluate` (squad_v2) + `datasets` | 0.4 / 2.19 | Официальные метрики EM/F1/NoAns |
| Данные | `pypdf` (PDF), `datasets` (SQuAD) | 4.0 / 2.19 | Инжест |

## Режимы работы ридера

| Режим | Скорость | Качество | Языки | Особенности |
|---|---|---|---|---|
| `extractive` | ~300 мс | F1 75.8 | EN | Точный span, калиброванный τ, аудит |
| `generative` | ~10 с | Containment 0.89 | EN+RU | Естественный ответ, цитаты [n] |
| `hybrid` | ~10 с | Containment 0.89 + UNVERIFIED 70% | EN+RU | Анти-галлюцинация, fallback на BERT |

## Воспроизводимость

- Все сиды зафиксированы в `config.py` (`seed=42` по умолчанию)
- `./run.sh eval` → полный прогон за 5–10 мин, результаты в `eval/results/*.csv`
- Зависимости закреплены в `requirements.txt`
- Модели скачиваются один раз (`scripts/download_models.py`), затем работа офлайн

## Ограничения (честно)

1. **Extractive reader только EN** — для RU используйте generative/hybrid
2. **SQuAD ≠ банковские документы** — перенос требует своей разметки для калибровки τ
3. **PDF без текстового слоя** — нужен OCR (не входит в baseline)
4. **Генерация на CPU** — секунды на ответ; для интерактива используйте extractive
5. **Таблицы в PDF** — теряются при извлечении pypdf
6. **τ не переносится** между доменами/конфигурациями — нужна перекалибровка
