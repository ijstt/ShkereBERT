# Инструкция по запуску ShkereBERT

Как поднять проект с нуля: окружение → модели → запуск (тесты, CLI, веб, оценка).
Технологии объяснены в `docs/ПОСОБИЕ.md`.

---

## 1. Требования
- Linux, Python 3.10+ (проверено на 3.12), ~8 ГБ свободной RAM, ~4 ГБ на диске под модели.
- CPU достаточно (GPU не требуется).

## 2. Окружение и зависимости
```bash
cd ~/ShkereBERT
python3 -m venv --system-site-packages .venv   # переиспользует системный torch, если есть
.venv/bin/pip install -r requirements.txt
```

## 3. Модели — что скачать

> Сеть на этой машине бывает нестабильной. Все команды докачивают с резюме — если оборвалось,
> просто **запусти ту же команду ещё раз**. Для sentence-transformers исключаем тяжёлые
> onnx/openvino варианты (`--exclude`), они нам не нужны.

Скачиваем в кэш HuggingFace (по умолчанию `~/.cache/huggingface`):

| Модель | Зачем | Размер | Обязательно |
|---|---|---|---|
| `sentence-transformers/all-MiniLM-L6-v2` | эмбеддер EN (baseline) | ~90 МБ | да |
| `deepset/tinyroberta-squad2` | экстрактивный ридер (EM/F1) | ~330 МБ | да |
| `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` | эмбеддер EN+**RU** (поиск на русском) | ~470 МБ | для RU |
| Qwen 2.5 3B Instruct GGUF (Q4_K_M) | генеративный ридер | ~2 ГБ | для generative/hybrid |
| `deepset/roberta-base-squad2` | более сильный ридер (сравнение) | ~500 МБ | опционально |
| `cross-encoder/ms-marco-MiniLM-L-6-v2` | reranker (усиление поиска) | ~90 МБ | будущее (Pool B) |
| `rajpurkar/squad_v2` (датасет) | замеры качества | ~45 МБ | для оценки |

Команды:
```bash
HF=.venv/bin/huggingface-cli

# Обязательные
$HF download sentence-transformers/all-MiniLM-L6-v2
$HF download deepset/tinyroberta-squad2

# Для русского языка (поиск)
$HF download sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2 --exclude "onnx/*" "openvino/*"

# Опционально / на будущее
$HF download deepset/roberta-base-squad2
$HF download cross-encoder/ms-marco-MiniLM-L-6-v2 --exclude "onnx/*" "openvino/*"

# Датасет для оценки
.venv/bin/python -c "from datasets import load_dataset; load_dataset('rajpurkar/squad_v2')"

# Генеративная модель Qwen (GGUF). Если уже есть локально — пропусти и просто укажи путь ниже.
$HF download Qwen/Qwen2.5-3B-Instruct-GGUF qwen2.5-3b-instruct-q4_k_m.gguf --local-dir models
# (по желанию — более сильная 7B):
# $HF download Qwen/Qwen2.5-7B-Instruct-GGUF qwen2.5-7b-instruct-q4_k_m.gguf --local-dir models
```

> На этой машине копии Qwen уже лежат в `/home/ijstt/home-ai-agent/models/` и `/home/ijstt/News/models/`.

## 4. Переменные окружения

```bash
# Путь к GGUF-модели генератора. НЕ обязателен, если файл лежит в ./models/
# (пайплайн по умолчанию берёт models/qwen2.5-3b-instruct-q4_k_m.gguf). Иначе укажи путь:
export SHKEREBERT_LLM=/home/ijstt/home-ai-agent/models/qwen2.5-3b-instruct-q4_k_m.gguf

# Включить русский поиск (многоязычный эмбеддер):
export SHKEREBERT_EMBED=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2

# После того как всё скачано — работать офлайн (быстрее и без обрывов сети):
export HF_HUB_OFFLINE=1 HF_DATASETS_OFFLINE=1 TRANSFORMERS_OFFLINE=1
```
`USE_TF=0` включать не нужно — уже прописано внутри пакета (иначе Keras 3 ломает импорт).

## 5. Запуск

Тесты:
```bash
PYTHONPATH=. .venv/bin/python -m pytest -q
```

CLI-чат (режимы: `extractive` | `generative` | `hybrid`):
```bash
# по документу (свой файл)
PYTHONPATH=. .venv/bin/python -m shkerebert.cli chat --file demo/machine_learning.txt --mode extractive

# генеративный ответ по русскому банковскому документу (нужен Qwen)
PYTHONPATH=. .venv/bin/python -m shkerebert.cli chat --file demo/bank_products_ru.txt --mode generative

# одиночный вопрос
PYTHONPATH=. .venv/bin/python -m shkerebert.cli ask --file demo/machine_learning.txt \
  --mode hybrid "When was the transformer introduced?"

# демо на SQuAD
PYTHONPATH=. .venv/bin/python -m shkerebert.cli chat --squad-n 300 --mode extractive
```

Веб-интерфейс (Streamlit):
```bash
PYTHONPATH=. .venv/bin/streamlit run app/streamlit_app.py
# откроется http://localhost:8501  — источник и режим ридера выбираются слева
```
Остановить веб: `Ctrl+C` в терминале, либо `pkill -f "streamlit run"`.

Оценка (пишет CSV/PNG в `eval/results/`):
```bash
PYTHONPATH=. .venv/bin/python -m eval.eval_retrieval --n 800     # Recall@k, MRR
PYTHONPATH=. .venv/bin/python -m eval.eval_e2e --n 1500          # EM/F1 + calibration/test + oracle
PYTHONPATH=. .venv/bin/python -m eval.ablations --n 400          # свипы chunk_size / top_k
```

## 6. Включить русский язык
1. Скачать многоязычный эмбеддер (см. §3).
2. `export SHKEREBERT_EMBED=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`
3. Запускать как обычно — теперь поиск работает по русским документам
   (`demo/bank_products_ru.txt`). Генерация Qwen двуязычна и без этого.

## 7. Если что-то не так (troubleshooting)
- **Скачивание обрывается (SSL / EOF)** → повтори ту же команду `huggingface-cli download`
  (докачивает с места обрыва). Для ST-моделей не забывай `--exclude "onnx/*" "openvino/*"`.
- **Зависает на сети при запуске** → выстави офлайн-переменные из §4 (модели уже в кэше).
- **Ошибка про Keras 3 / TF** → должно лечиться само (`USE_TF=0` в пакете); если запускаешь
  отдельный скрипт мимо пакета — добавь `USE_TF=0` в окружение.
- **`generative`/`hybrid` не стартует** → проверь `SHKEREBERT_LLM` (путь к .gguf) и что
  `llama-cpp-python` установлен (`.venv/bin/pip show llama-cpp-python`).
- **Медленно** → это CPU. Экстрактивный режим быстрый (~0.2 с), генеративный медленнее
  (несколько секунд на ответ) — норма для локальной LLM на CPU.
